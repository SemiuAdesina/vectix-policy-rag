# Client Communication Guidelines

**Policy ID:** VL-OPS-012  
**Effective Date:** January 1, 2026  
**Focus:** SLA response times (1hr for critical issues).  
**Version:** 2.0.1

## 1. Response Times

- **Critical (System Down):** 1 hour response time.
- **High (Feature Bug):** 4 hour response time.
- **Normal (General Inquiry):** 24 hour response time.

## 2. Tone and Style

Communications should be authoritative yet helpful. Avoid overly technical jargon when speaking with pharmacy managers; use clear, simple English.

## 4. Appendix: Definitions

| Term | Definition |
|------|------------|
| **SLA** | Service Level Agreement; committed response or resolution time. |
| **Critical** | Severity for system-down or equivalent; 1-hour response time. |
| **Response time** | Time from report to first substantive response (e.g., acknowledgment, triage). |
| **System Down** | Production system unavailable to client; highest priority. |
| **High** | Severity for feature bug or major impact; 4-hour response time. |
| **Feature Bug** | Defect in functionality; not full outage. |
| **Normal** | General inquiry or low severity; 24-hour response time. |
| **General Inquiry** | Question, request, or non-urgent issue. |
| **Authoritative** | Confident, accurate, and aligned with company position. |
| **Helpful** | Aimed at resolving the client's need. |
| **Jargon** | Overly technical terms that may confuse non-technical clients. |
| **Pharmacy managers** | Client role often non-engineer; prefer clear, simple English. |
| **Tone** | Manner of written and verbal communication. |
| **1 hour** | Target response time for critical (System Down). |
| **24 hour** | Target response time for normal inquiries. |
